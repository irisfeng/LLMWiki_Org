# llmWiki_Org 功能与 UI 优化设计

> 日期：2026-04-17
> 目标：让用户方便易用，上传文档后知识沉淀准确，AI 问答好用
> 用户：5-20 人技术团队内部工具

## 产品北极星

**易用 → 准确 → 好问**

- 新成员 3 秒理解系统是干什么的
- 上传文档零摩擦，知识自动沉淀
- 发现错误能快速修正
- AI 问答能追问、能溯源、值得信任

## 1. 主题系统（亮/暗双模式）

### 方案

在现有 Element Plus 上叠加 CSS 变量主题层。

### 实现

- 用 `@vueuse/core` 的 `useDark()` + `useToggle()` 管理主题状态
- 定义 CSS 变量集合（`--bg-primary`, `--bg-secondary`, `--text-primary`, `--text-muted`, `--border-color`, `--accent` 等）
- 亮色和暗色各一套变量值，挂在 `html.dark` 类上切换
- Element Plus 通过 `el-config-provider` 的 `namespace` 配合其内置 dark mode 支持
- `AppLayout.vue` 顶栏加主题切换按钮
- 所有现有自定义 CSS 中的硬编码颜色值迁移到 CSS 变量

### 变量定义

```css
:root {
  --bg-primary: #ffffff;
  --bg-secondary: #f9fafb;
  --bg-card: #ffffff;
  --text-primary: #111827;
  --text-secondary: #6b7280;
  --text-muted: #9ca3af;
  --border: #e5e7eb;
  --accent: #3b82f6;
  --accent-soft: #eff6ff;
}

html.dark {
  --bg-primary: #0f172a;
  --bg-secondary: #1e293b;
  --bg-card: #1e293b;
  --text-primary: #f1f5f9;
  --text-secondary: #94a3b8;
  --text-muted: #64748b;
  --border: #334155;
  --accent: #60a5fa;
  --accent-soft: #1e3a5f;
}
```

## 2. 导航改造

### 当前问题

左侧菜单项平铺（首页/文档库/信息源/实体/概念），搜索只在首页内容区，功能入口不突出。

### 新结构

```
顶栏（固定）：
  左：Logo "团队知识库"
  中：全局搜索框（Cmd+K 唤起）
  右：主题切换 🌙/☀️ | 登出

左侧导航（可折叠）：
  🏠 首页
  💬 AI 问答        ← 核心功能提升位置
  📚 知识库          ← 点击展开子菜单
     ├ 全部
     ├ 信息源
     ├ 实体
     └ 概念
  📁 文档管理        ← 原"文档库"
```

### 全局搜索

- 顶栏搜索框，`Cmd+K` / `Ctrl+K` 快捷键聚焦
- 输入时实时下拉：前 5 条匹配的 wiki 页面（标题 + 类型标签 + 摘要片段）
- 回车跳转搜索结果页
- 自然语言问句（检测到问号或长句）→ 提示"在 AI 问答中提问"，点击跳转 Chat 并带入

## 3. 首页重做

### 当前问题

90+ 条记录平铺表格，新用户不知道系统是什么、怎么用。

### 新首页布局

**区域 1：欢迎 + 搜索/提问框**

- 标题："团队知识库"
- 副标题："上传文档，自动结构化，随时提问"
- 大搜索框，placeholder 轮换引导语（"搜索知识或直接提问..."）
- 搜索框下方 3 个快捷按钮：「上传文档」「AI 问答」「浏览知识库」

**区域 2：统计概览**

- 横排 4 个精致卡片：信息源数 / 实体数 / 概念数 / 文档数
- 带图标，点击跳转对应筛选页
- 用 CSS 变量适配亮暗模式

**区域 3：最近更新**

- 按天分组的动态流（今天 / 昨天 / 更早）
- 每条：类型标签 + 标题（可点击） + 摘要前 80 字 + 相对时间
- 最多 15 条，底部"查看全部"
- 不再是无限表格

**区域 4：按类型浏览**

- 3 个类型卡片横排：信息源 / 实体 / 概念
- 每个卡片内展示最新 3 个页面标题
- 点击标题直接跳转，点击卡片头部跳转列表

## 4. 聊天增强

### 4a. 多轮追问

**后端改进（`query.py`）：**

- 当 history 超过 4 轮时，用 LLM 对前面的对话生成一句话摘要
- prompt 构成：system（角色+知识库说明）+ 历史摘要 + 最近 3 轮完整对话 + 检索上下文 + 当前问题
- 控制总 token 量在 4000 以内（不含流式输出）

**前端改进（`ChatView.vue`）：**

- 空会话展示 3-4 个推荐问题卡片（基于知识库热门内容），点击直接发送
- `Enter` 发送，`Shift+Enter` 换行
- 流式回答时"正在思考..."骨架动画
- 错误状态更友好："暂时无法回答，请稍后重试" + 重试按钮

### 4b. 知识探索面板

**聊天页改为可选三栏布局：**

- 左栏（240px，可折叠）：会话历史列表（当前 drawer 改为常驻侧栏）
- 中栏（自适应）：对话流
- 右栏（300px，可折叠，默认收起）：知识探索面板

**右侧知识探索面板内容：**

- "引用来源"区域：
  - 当前回答的 referenced_pages 展开为卡片
  - 每张：页面标题 + 类型标签 + 匹配的 chunk 摘要（前 60 字）
  - 点击新标签打开 wiki 页面
- "相关页面"区域：
  - 基于引用页面的 wikilinks 展开 1 度关联
  - 列表形式：标题 + 类型，点击跳转
- 移动端（<768px）：右栏隐藏，通过浮动按钮弹出底部面板

### 4c. 新增后端接口

- `GET /api/wiki/pages/{slug}/related` — 返回页面的 wikilink 关联页面列表（1 度）
- 用于聊天知识探索面板的"相关页面"区域

## 5. Wiki 轻量编辑

### 需求

修正 LLM 自动生成的错误内容，不需要从零建页面。

### 实现

**前端：**

- WikiPageView 页面顶部加"编辑"按钮
- 点击切换为编辑模式：
  - 上方：Markdown textarea（等宽字体，自动撑高）
  - 下方：实时预览（复用 MarkdownRenderer 组件）
- "保存"和"取消"按钮
- 保存时调用 API，成功后切回阅读模式

**后端：**

- `PUT /api/wiki/pages/{slug}` — 更新页面 content
  - 请求体：`{ content: string, edited_by?: string }`
  - 更新 content、updated_at
  - 在 frontmatter 中记录 `last_edited_by` 和 `last_edited_at`
  - 异步触发 re-chunk + re-embed（Celery 任务）
- 不做版本历史（YAGNI，后续需要时再加）

## 6. 全局体验细节

### 加载状态

- 所有列表/内容区用骨架屏（skeleton）替代 `v-loading` 转圈
- Element Plus 的 `el-skeleton` 组件

### 空状态

- 每个空列表都有引导文案 + 行动按钮
- 首页无内容："还没有知识，上传第一份文档开始吧" + 上传按钮
- 搜索无结果："没有找到相关内容，试试在 AI 问答中提问？" + 跳转按钮
- 聊天空会话：推荐问题卡片

### 面包屑

- Wiki 页面：首页 > 知识库 > 概念 > RAG 检索原理
- 文档管理：首页 > 文档管理

### 响应式

- 768px 以下：左侧导航折叠为汉堡菜单
- 聊天三栏布局在移动端退化为单栏
- 首页统计卡片 2x2 排列

## 7. 不做的事情（明确排除）

- 不换组件库（保持 Element Plus）
- 不做完整 wiki 编辑器（只做 textarea 编辑）
- 不做版本历史
- 不做多用户权限系统
- 不做知识图谱可视化（只做列表级关联展示）
- 不做分析仪表盘（本轮不含）

---

## 8. 实现顺序建议

1. **主题系统 + CSS 变量** — 基础设施，后续所有 UI 改动都依赖它
2. **导航改造** — 全局骨架变了，其他页面才有锚点
3. **首页重做** — 用户第一印象，优先级最高的体验问题
4. **聊天增强** — 核心功能价值
5. **Wiki 轻量编辑** — 知识准确性保障
6. **全局体验细节** — 收尾打磨

每个阶段独立交付、独立部署、独立验证。
