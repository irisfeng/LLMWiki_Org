# 团队知识库 · 使用手册

> 一份能在 5 分钟内学会的操作指南。
> 访问地址：`http://124.222.82.73:8080`

---

## 一、它是什么

把团队的文档（PDF / Word / PPT / 网页 / 文本）丢进去，系统会自动：

1. **拆分整理** — 抽出里面的概念、实体、要点
2. **建立链接** — 同一概念在不同文档之间自动互联
3. **支持问答** — 你直接用自然语言问，AI 从知识库里找答案 + 给出引用来源

简单说：**一个会读你团队所有文档的 AI 助手**。

## 二、3 分钟上手

### 1️⃣ 登录
浏览器打开 `http://124.222.82.73:8080` → 输入账号密码。

### 2️⃣ 上传第一份文档
左下角侧边栏 **"上传文档"** 按钮 → 拖拽文件进去（PDF / DOCX / PPTX / MD / URL 都行）→ 上传后系统会自动处理（约 1–3 分钟）。

### 3️⃣ 直接问 AI
任意页面右下角点 **✦** 浮动按钮（或按 **⌘ J / Ctrl J**）打开 AI 抽屉 → 输入问题 → 看到答案 + 引用的文档来源。

> 💡 也可以从首页"问一个问题或搜索"框直接开始。

## 四、4 种内容类型（重要）

知识库里所有页面都是这 4 类之一，颜色和图标不同便于识别：

| 类型 | 标签颜色 | 含义 | 例子 |
|------|---------|------|------|
| **信息源** S | 绿色 | 你上传的原始文档 | "RAG Paper 2020.pdf" |
| **实体** E | 橙色 | 提到的人/公司/产品 | "Andrej Karpathy"、"Anthropic" |
| **概念** C | 蓝色 | 跨文档抽象的想法 | "RAG"、"Context Engineering" |
| **分析** A | 紫色 | 团队得出的结论 | "Q2 知识盲区梳理" |

> 实体/概念/分析都是 AI 自动从你上传的信息源里抽出来的。

## 五、最常用的 5 个场景

### 场景 1：找一份文档讲了什么
1. 进 **知识库** → 点 **信息源** tab
2. 找到那份文档卡片 → 点开
3. 看到：标题 + 衬线副标题 + **PDF/DOCX hero card** + AI 摘要 + 关键要点 + 反向链接
4. 想看原文 → 右上 **下载原文**

### 场景 2：搜索某个概念
- 侧边栏搜索框（或按 **⌘ K**）→ 输关键词回车

### 场景 3：问一个跨文档的问题
- **⌘ J** 打开 AI 抽屉，例如：
  - "我们对 RAG 的看法是什么？"
  - "Q2 战略备忘录里有哪些关键决定？"
- 答案下方会列出 **参考来源**，点击可跳到对应文档

### 场景 4：上传一份新文档
- 侧边栏 **"上传文档"** → 选文件 / 拖拽 / 输入 URL → 等待状态变成 "已处理"
- 系统自动从中抽取实体和概念，与已有页面建立链接

### 场景 5：检查知识库健康度
- 进 **健康检查** → 看综合评分（90+ 优秀 / 75+ 良好 / 60+ 一般 / <60 需关注）
- 看 4 色严重程度卡片 → 点 **修复** 按钮按建议处理

## 八、常见问题

**Q: 上传文档后多久能用？**
A: 通常 1–3 分钟。在 **文档** 页可以看处理状态：待处理 → 处理中 → 已处理。

**Q: 支持哪些文件格式？**
A: PDF / DOCX / PPTX / XLSX / MD / TXT / HTML / 网页 URL。

**Q: AI 答案不准怎么办？**
A: 先看引用来源，可能是源文档本身有问题。也可以在 **健康检查** 看是否有"内容矛盾"提醒。

**Q: 我能删自己上传的文档吗？**
A: 进 **文档** 页 → 右侧 **删除** 按钮。删除时可选是否级联删除从中抽出的 wiki 页面。

**Q: 一段话从哪里来的？**
A: 鼠标停在 wiki 页的内容上，里面的 `[[xxx]]` 链接点进去能找到原始信息源。
